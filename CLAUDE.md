# FoodScribe — CLAUDE.md

## Project purpose
FoodScribe converts free-text meal descriptions into USDA-grounded nutrient profiles,
including food category classification and statistical macro analysis.

## Four-stage pipeline
1. **LLM parsing** (`foodscribe/llm/`) — parse meal text → structured JSON food items
2. **Semantic retrieval** (`foodscribe/retrieval/`) — embed item name with MPNet,
   cosine-search USDA descriptions → return best-match `fdc_id`
3. **Nutrient + category lookup** (`foodscribe/nutrients/`) —
   pull exact USDA nutrient row AND food category/subcategory by `fdc_id`
4. **Stats + visualisation** (`foodscribe/analysis/`) —
   compute per-meal and aggregate statistics; plot distributions grouped by category

## Data files (NOT committed to git — generated by scripts/build_data.py)
- `data/food_embeddings_mpnet.npy`  shape (N, 768), float32, L2-normalised
- `data/food_metadata.csv`          columns: fdc_id, description, food_category, data_type
- `data/foods_wide.csv`             columns: fdc_id, <nutrient_name>...
- `data/food_categories.csv`        columns: fdc_id, description, category, subcategory, data_type

## Raw USDA data (source — NOT generated)
- `../USDA_data/foundation_long.csv`  — Foundation foods, long format
- `../USDA_data/legacy_long.csv`      — SR Legacy foods, long format
- `../USDA_data/survey_long.csv`      — Survey/FNDDS foods, long format
- All three share the same schema: fdc_id, data_type, description, food_category, nutrient_name, amount, unit_name, ...

## Building data files
Run once before using the app:
```bash
python scripts/build_data.py --usda-dir ../USDA_data/ --data-dir data/
# Add --skip-embeddings to skip the slow sentence-transformer step
```

## food_categories.csv — how it is built
`scripts/build_data.py` reads the long CSVs, deduplicates by fdc_id, and infers:
- `category` from the `food_category` column (empty for Survey → "Unknown")
- `subcategory` from the first comma-segment of the description
- `data_type` from the `data_type` column, normalised to foundation/sr_legacy/survey_fndds

## Key design rules
- The retriever does NOT train or fine-tune any model.
  It only runs inference with `sentence-transformers/all-mpnet-base-v2`.
- Nutrient values are USDA ground truth — never predicted by ML.
- Food categories are USDA-assigned metadata — never inferred by ML.
- LLM provider is configurable; default is `anthropic` (`claude-haiku-4-5-20251001`).
  Supported: `anthropic`, `openai`, `deepseek`, `gemini`.
- Anthropic is the recommended default — it is the same model family used to develop this tool.
- All secrets come from `.env`; never hard-code keys.
- Return types are typed dataclasses / TypedDicts throughout.
- CLI entry-point: `foodscribe parse "<meal text>"`

## Running the tool
```bash
pip install -e .
foodscribe parse "2 scrambled eggs with toast and orange juice"
foodscribe parse --provider anthropic "bowl of oatmeal with banana"
foodscribe parse --provider deepseek "grilled salmon with brown rice"
foodscribe analyse --meal "chicken tikka masala with naan"
foodscribe categories --fdc-id 171705
```
